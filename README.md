# Data Dimensionality Reduction and Spatial Analysis

---

## __О проекте__

Данный проект — реализация анализа высокоразмерных данных на языке C.

Проект выполняет загрузку и стандартизацию датасета Wine, уменьшает размерность признаков методом PCA, кластеризует полученную двумерную проекцию методами K-Means и DBSCAN, а также сравнивает пространственные запросы, выполненные полным перебором и через k-d tree.
---

## __Что реализовано__

- чтение датасета Wine из CSV-файла;
- хранение данных в собственной структуре `Matrix`;
- операции линейной алгебры;
- стандартизация признаков;
- PCA для перехода из 13 признаков в 2D;
- визуализация PCA-проекции;
- кластеризация методом `k-means`;
- кластеризация методом `DBSCAN`;
- поиск ближайшего соседа полным перебором;
- поиск точек в радиусе полным перебором;
- построение `k-d tree`;
- поиск близжайших соседей через `k-d tree`;
- поиск точек в радиусе через `k-d tree`;
- сравнение корректности и времени работы brute force и `k-d tree`;
- SVG-визуализация всех результатов.

## __Датасет__

В проекте используется Wine Dataset.

Характеристики данных:

```text
Количество объектов: 178
Количество признаков: 13
Количество классов: 3
```

Первый столбец содержит метку класса вина. Остальные 13 столбцов содержат числовые химические признаки.

Метки классов сохраняются отдельно и не участвуют в PCA. Они используются только для проверки и визуализации.

---

## __Стандартизация данных__

Перед применением PCA каждый признак стандартизируется:

```text
x_new = (x - mean) / standard_deviation
```

Это нужно, потому что признаки Wine имеют разные масштабы. Без стандартизации признаки с большими числовыми значениями сильнее влияли бы на PCA и кластеризацию.

---

## __PCA__

PCA используется для уменьшения размерности данных с 13 признаков до 2 координат.

В проекте реализованы:

1. построение ковариационной матрицы;
2. поиск главной компоненты степенным методом;
3. дефляция ковариационной матрицы;
4. поиск второй главной компоненты;
5. проекция данных на плоскость `PC1` и `PC2`.

Пример результата для Wine:

PCA explained variance:
PC1: 36.20%
PC2: 19.21%
Together: 55.41%


Это означает, что первые две компоненты сохраняют около 55% дисперсии исходного 13-мерного набора данных.

---

## __K-Means__

Метод `k-means` используется для кластеризации PCA-проекции.

Реализованы основные этапы алгоритма:

- выбор начальных центроидов;
- назначение каждой точки ближайшему центроиду;
- пересчёт центроидов как среднего значения точек кластера;
- остановка, если точки больше не меняют кластеры.

Для Wine используется `k = 3`, так как в исходном датасете три класса вин.

Пример вывода:

K-means clusters:
Iterations: 5
Cluster 1: 61 points
Cluster 2: 67 points
Cluster 3: 50 points

## __K-D Tree и Brute Force__

В проекте реализованы два способа пространственного поиска.

### Brute Force

Полный перебор проверяет расстояние от точки запроса до каждой точки набора данных.

Плюсы:

- простая реализация;
- удобно использовать как эталон корректности.

Минусы:

- время работы линейно зависит от количества точек.

### K-D Tree

`k-d tree` — это бинарное дерево для хранения точек в пространстве.

В проекте дерево используется для:

- поиска ближайшего соседа;
- поиска всех точек внутри заданного радиуса.

Сравнение с brute force позволяет проверить, что дерево возвращает тот же результат.

Пример вывода:

```
Spatial queries:
Nearest brute force: Wine 11
Nearest k-d tree:    Wine 11
Nearest equal: yes
Range radius 1.00: brute=22, tree=22, equal=yes
```
Для поиска близжайшего соседа оба метода нашли один и тот же объект (Wine 11). Для range query при радиусе 1.0 оба метода нашли одинаковое множество из 22 объектов. Это подтверждает корректность реализации k-d tree.

## __DBSCAN__

`DBSCAN` — алгоритм кластеризации, который группирует точки по плотности.

В проекте DBSCAN использует `range query` из `k-d tree` для поиска соседей точки.

Основные параметры:

- `eps` — радиус поиска соседей;
- `min_points` — минимальное количество точек для образования плотной области.

Алгоритм выделяет:

- обычные кластеры;
- шумовые точки, которые не принадлежат ни одному кластеру.

Результат сохраняется в SVG-файл.
